    ## Required configuration

    ### Overview

    The configuration file, luminaConfig, acts as a central registry where each component is defined with its specific settings. This setup ensures that components are modular, easily configurable, and maintainable across Lumina.

    ### Configuration Structure

    Each component in the luminaConfig object is defined by a specific structure:

```js

TComponentConfig = {
    [key: string]: {
        component: (data: any) => React.JSX.Element | null;
        config: TConfig;
    };
};


    component: {
    component: componentType,
    config: componentConfig,
    },
```

_- **componentName**: This is a unique identifier for the component in the configuration file.
_-**component**: This refers to the React component itself.
**config**: This object contains the configuration details specific to the component, such as default props, style overrides, or other settings that customize the component's behavior.
